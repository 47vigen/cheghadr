# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Cheghadr? (چه‌قدر؟) is a bilingual personal net worth tracker (English default, Persian supported) — a Telegram Mini App + standalone web app.

**Stack:** Next.js 16 (App Router, Turbopack) · TypeScript 5 (strict) · tRPC v11 · Prisma 7 + Neon Serverless PostgreSQL · NextAuth v5 (Telegram provider) · HeroUI v3 · Tailwind CSS v4 · Jotai · TanStack React Query v5 · next-intl (English + Persian)

## Commands

```bash
pnpm dev          # Dev server on port 3000 (Turbopack + Node.js debugger)
pnpm build        # Prisma codegen → Next.js build
pnpm check        # typecheck + lint (run before pushing)
pnpm typecheck    # next typegen && tsc --noEmit
pnpm lint         # Biome lint check
pnpm lint:fix     # Fix lint issues (unsafe mode)
pnpm biome:fix    # Biome check --write (lint + format combined)
pnpm format:write # Apply Biome formatting only
pnpm test         # Vitest (single pass)
pnpm test:watch   # Vitest watch mode
pnpm db:push      # Push Prisma schema to DB (no migration)
pnpm db:migrate   # Prisma migrate dev
pnpm db:studio    # Prisma Studio UI
pnpm db:seed      # Run seed script
pnpm swagger      # Regenerate Ecotrust API types from OpenAPI spec + format
```

## Architecture

### Route structure

- `src/app/(app)/` — Protected routes: `/dashboard`, `/prices`, `/alerts`, `/assets/add`, `/calculator`
- `src/app/login/` — Telegram Web App login
- `src/app/api/trpc/[trpc]` — tRPC endpoint
- `src/app/api/auth/[...nextauth]` — NextAuth routes
- `src/app/api/cron/prices` and `cron/portfolio` — Price/portfolio snapshot jobs (production: invoked by **cron-job.org**, not Vercel Cron — see `docs/cron-scheduling.md`)

### Backend layer (`src/server/`)

- `api/routers/` — tRPC procedures: `alerts`, `assets`, `portfolio`, `prices`, `user`
- `auth/` — NextAuth config with Telegram credentials provider
- `cron/` — Cron job handlers that call the Ecotrust external price API and write snapshots

### Data flow

1. External price data comes from `NEXT_PUBLIC_ECOTRUST_API_URL` (ecotrust.ir), fetched by cron jobs and stored as `PriceSnapshot` rows
2. tRPC routers serve client queries; React Query caches results in the browser
3. Partial React Query cache is persisted to localStorage (see `src/trpc/persister.ts`) — only successful queries, and **user-scoped routers** (`alerts`, `assets`, `portfolio`, `user`) are **excluded** from persistence. Bump `TRPC_QUERY_PERSIST_BUSTER` to invalidate all clients' stored cache.

### Auth proxy

Auth runs in `src/proxy.ts` (Next.js 16 proxy, Node.js runtime) — **not** Edge middleware. All `/api/*` routes pass through without auth checks.

### UI components

- `src/components/ui/` — Shared primitives built on HeroUI v3. For new primitives, follow existing patterns here; **do not use shadcn**.
- Feature UI is organized by domain: `components/assets/`, `portfolio/`, `prices/`, `alerts/`, `calculator/`

### i18n

English (`en`) is the default locale in config and for new `User.preferredLocale` rows; Persian (`fa`) is fully supported with RTL. The web client picks `en` or `fa` from Telegram `language_code` or `navigator.language` (`LocaleProvider`). Translations live in `messages/*.json` and `src/i18n/`.

### Database models

Key Prisma models: `User` (Telegram profile), `Portfolio`, `UserAsset` (holdings per portfolio, composite unique on userId+symbol+portfolioId), `PriceSnapshot` (JSON blob, indexed by date), `PortfolioSnapshot` (historical totals), `Alert` (PRICE/PORTFOLIO type, ABOVE/BELOW direction).

### Generated code

- `src/modules/API/Swagger/ecotrust/gen/` — Auto-generated from OpenAPI spec via Kubb. Do not edit manually; run `pnpm swagger` to regenerate.

## Development Gotchas

- **Skip login locally**: Set `DEV_TELEGRAM_USER_ID="<your-telegram-id>"` in `.env.local` to bypass Telegram auth entirely. Production ignores this var.
- **Turbopack cache corruption**: If dev server panics with "Failed to restore task data", delete `.next/` and restart.
- **Stale dev lock**: If `pnpm dev` fails with "Unable to acquire lock at .next/dev/lock", kill orphan `next-server` processes and remove `.next/dev/lock`.
- **Telegram Login Widget**: Shows "Bot domain invalid" in local dev — expected, because Telegram Widget requires a domain registered with BotFather.
- **Cron endpoints**: `/api/cron/prices` and `/api/cron/portfolio` require `Authorization: Bearer $CRON_SECRET` (set on **cron-job.org** or curl). Vercel Cron is not used (`vercel.json` has no `crons`). See `docs/cron-scheduling.md`. The Ecotrust API may be unreachable from some networks; cron 500 on fetch failure is expected in dev.
- **`--inspect` flag**: `pnpm dev` includes `--inspect`. "Starting inspector failed" warning is harmless if port 9229 is in use.
- **`SKIP_ENV_VALIDATION=1`**: Bypasses `@t3-oss/env-nextjs` strict validation (useful for builds with incomplete env vars).

## Environment Variables

Required: `DATABASE_URL`, `DIRECT_URL`, `NEXTAUTH_SECRET`, `NEXTAUTH_URL`, `TELEGRAM_BOT_TOKEN`, `CRON_SECRET`, `NEXT_PUBLIC_ECOTRUST_API_URL`, `NEXT_PUBLIC_TELEGRAM_BOT_USERNAME`

Optional (dev only): `DEV_TELEGRAM_USER_ID`

See `src/env.js` for validation schema.

## Code Conventions

- **Linter/formatter**: Biome (not ESLint/Prettier). 2-space indent, single quotes, trailing commas, 80-char line width.
- **Commits**: Conventional Commits enforced by commitlint. Types: `chore`, `ci`, `docs`, `feat`, `fix`, `refactor`, `revert`, `style`.
- **Git hooks**: Husky runs lint-staged (pre-commit), commitlint (commit-msg), `pnpm typecheck` (pre-push).
- **Path alias**: `@/*` maps to `src/*`.
- **State**: Jotai for client state, React Query for server state. Don't introduce other state libraries.

## Testing

Tests live alongside source in `src/**/__tests__/*.test.ts{,x}`. Coverage thresholds: 70% lines/functions, 60% branches. Coverage is collected from `src/lib/**`, `src/server/**`, `src/types/**`, `src/trpc/**`, `src/hooks/**`. Generated API code (`src/modules/API/**`) is excluded.
